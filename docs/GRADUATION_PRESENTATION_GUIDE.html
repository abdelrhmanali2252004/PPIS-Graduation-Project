3. نبذة عن المشروع والفكرة الأساسية

ما هو فكرة TECH؟

فكرة TECH منصة ويب عربية (RTL) تساعد رواد الأعمال في مصر — وخاصة في الصعيد — على تحويل فكرة المشروع إلى خطة عمل قابلة للتنفيذ، من خلال مسار رقمي من 5 خطوات يجمع بين الذكاء الاصطناعي، أبحاث السوق، دراسة الجدوى، الهوية البصرية، ولوحة تحكم استراتيجية لمتابعة الأداء.

المشكلة التي نحلها

  • كثير من أصحاب المشاريع الصغيرة يبدأون بدون دراسة جدوى منظمة أو بيانات سوق موثقة.
  • تكلفة ووقت إعداد دراسات الجدوى التقليدية مرتفعان وغير متاحين للجميع.
  • غياب أداة عربية متكاملة تجمع: بحث سوق + أسئلة منظمة + تقرير AI + شعار + متابعة مالية.

الحل الذي نقدمه

  • منصة واحدة تُرشد المستخدم من أول الفكرة حتى لوحة التحكم التنفيذية.
  • توليد دراسة جدوى بالذكاء الاصطناعي مع إمكانية دمج ملف PDF لأبحاث السوق (RAG).
  • توليد شعار وهوية بصرية بالـ AI حسب بيانات المشروع.
  • لوحة تحكم بمؤشرات مالية، تنبيهات ذكية، ورسوم بيانية مستخرجة من دراسة الجدوى.
  • لوحة إدارة للمشرفين لمتابعة المستخدمين والمشاريع والطلبات.

الجمهور المستهدف

رواد أعمال، طلاب مشاريع تخرج، أصحاب مشاريع صغيرة ومتوسطة في السوق المصري — مع تركيز على واقع السوق المحلي.

5. الـ Features (المميزات) بالتفصيل

5.1 الصفحة الرئيسية (Landing)

  • عرض تسويقي للمنصة: Hero، الرؤية، الخدمات، دعوة للبدء، تواصل.
  • تبديل اللغة (عربي/إنجليزي) والثيم (فاتح/داكن).
  • تسجيل دخول / إنشاء حساب من الهيدر.

5.2 المصادقة والحساب

  • تسجيل مستخدم جديد (اسم، بريد، هاتف، كلمة مرور).
  • تسجيل دخول بـ JWT — التوكن يُحفظ ويُرسل مع كل طلب API.
  • أدوار: user | admin | specialist.
  • إعدادات البروفايل: تعديل الاسم، OTP لإضافة هاتف ثانوي، تغيير كلمة المرور، استعادة كلمة المرور.

5.3 مسار المشروع — 5 خطوات (App Wizard)

1
مركز أبحاث السوق
رفع ملف PDF لدراسة سوق موجودة، أو طلب بحث سوق من المنصة. يُنشأ Project ID ويُربط بالجلسة.
2
تفاصيل المشروع (Wizard)
24 سؤالاً منظماً في 5 مراحل: الهوية، السوق والعملاء، التشغيل، المالية، المخاطر والتوقعات.
3
مخرجات دراسة الجدوى
AI يولّد تقريراً عربياً شاملاً: ملخص تنفيذي، سوق، منافسين، تشغيل، تسويق، تكاليف، أرباح، مخاطر، توصيات، خطة 90 يوم.
4
الهوية البصرية
جمع بيانات البراند → Groq يولّد Prompt → Pollinations يولّد الشعار → حفظ في قاعدة البيانات.
5
لوحة التحكم الاستراتيجية
KPIs مالية، رسوم بيانية، تنبيهات ذكية، تبويبات للتوقعات والسوق والتشغيل، تصدير PDF للتقرير.

5.4 لوحة المستخدم (User Dashboard)

  • عرض كل مشاريع المستخدم مع المرحلة الحالية.
  • فتح مشروع والاستكمال من آخر خطوة.
  • لوحة محتوى تفصيلية لكل مشروع مكتمل (Step 5).
  • إعدادات الحساب والمظهر.

5.5 لوحة الإدارة (Admin Dashboard)

  • إدارة المستخدمين — عرض كل الحسابات.
  • إدارة المشاريع — عرض المشاريع، المرحلة، المالك، فتح لوحة المشروع.
  • إدارة الطلبات — طلبات التواصل مع المختص (بحث سوق، استشارة...).

5.6 مميزات تقنية إضافية

  • RAG: دمج محتوى PDF أبحاث السوق في prompt دراسة الجدوى لزيادة الدقة.
  • تنبيهات ذكية مبنية على مؤشرات مستخرجة من التقرير.
  • جلسة منتهية — معالجة 401 وتسجيل خروج تلقائي.
  • دعم RTL كامل + خط Cairo + تصميم متجاوب.

8. أسئلة متوقعة في Q&A (وجواب مختصر)

أسئلة Business

السؤالإجابة مقترحة
إيه الفرق بينكم وبين دراسة جدوى تقليدية؟منصة رقمية فورية بالـ AI، أرخص وأسرع، مع متابعة مستمرة عبر Dashboard.
مين الجمهور المستهدف؟رواد أعمال في مصر، مشاريع صغيرة، طلاب — مع بيانات سوق محلية.
إزاي بتكسبوا فلوس؟[حدّدوا نموذج العمل: اشتراكات / خدمات مدفوعة / عمولة استشارات]
إيه الميزة التنافسية؟مسار متكامل 5 خطوات + عربي + RAG + شعار + لوحة استراتيجية في مكان واحد.

أسئلة Technical

السؤالإجابة مقترحة
ليه React و Redux؟واجهة تفاعلية معقدة، حالة موزعة (Auth, Projects, Feasibility...) — Redux ينظمها.
ليه MongoDB؟مرونة في هيكل بيانات المشروع، إجابات Wizard، تقارير JSON من AI.
إزاي بتتعاملوا مع الأمان؟JWT، bcrypt لكلمات المرور، OTP للعمليات الحساسة، Middleware للتحقق.
إيه الـ RAG؟نستخرج نص PDF أبحاث السوق، نقسّمه، نبحث فيه، ونضيفه لـ prompt الجدوى.
ليه Proxy للشعار؟Pollinations ترفض طلبات المتصفح المباشرة — الباك إند يجلب الصورة server-side.
إيه الـ API Endpoints الرئيسية؟/user/login, /project/step1-4, /project/step3, /project/save-logo, /request/create-request...
إزاي بتتعاملوا مع انتهاء الجلسة؟Interceptor على Axios — عند 401 أو session expired يُعاد توجيه المستخدم.

9. قرارات تقنية مهمة (للشرح أمام اللجنة)

  • فصل Frontend و Backend — REST API واضح يسهّل الصيانة والتوسع.

  • حفظ Project ID في localStorage لاستمرار الجلسة بين الخطوات.
  • Typescript في Frontend لسلامة الأنواع مع عقد API.
  • دعم i18n و Theme من البداية لتجربة مستخدم أوسع.
  • تصدير PDF من Dashboard باستخدام html2canvas لدعم العربية في التقارير.
